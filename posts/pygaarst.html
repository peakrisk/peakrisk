time to find gdal¶

Installing with pip failing due to missing cpl_port.h

So, plan B, install into Ubuntu and tell virtualenv to use system stuff.

You can do this when you create a virtualenv as follows:

mkvirtualenv --system-site-packages foo

This creates an empty file: .virtualenvs/foo/lib/python3.4/no-global-site-packages.txt

Renaming or deleting this will make the virtualenv pick up packages installed in the system python.
